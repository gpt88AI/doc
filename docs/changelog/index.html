2026-07-21

文档站补齐多语言入口和核心接入链路，面向全球开发者开放 13 种语言，并同步完善多语言 SEO、预渲染和未翻译页面回退策略。

  • 新增

    新增 13 种语言入口

    文档站现在支持简体中文、English、繁體中文、西班牙语、巴西葡萄牙语、法语、德语、阿拉伯语、日语、印尼语、俄语、韩语和越南语。语言入口使用独立 URL 前缀，例如 /en/、/pt-br/、/id/、/vi/。

  • 新增

    核心接入页面完成本地化

    首页、产品概览、快速开始、认证与计费、FAQ 已接入多语言文案。内容优先覆盖获取 API Key、Base URL、首次请求、计费和模型可用性等高转化信息，减少海外用户首次接入时的语言障碍。

  • 优化

    增加语言方向与本地化排版支持

    语言配置统一维护原生语言名称、URL 前缀、hreflang、文本方向和短名称。阿拉伯语页面输出 RTL 方向，其它语言页面输出对应的 lang 属性，避免只翻译按钮而忽略页面语义。

  • 优化

    多语言 SEO 与 GEO 资产同步生成

    为多语言核心页面生成独立 canonical、hreflang、x-default、sitemap 和静态 HTML。未完成全文翻译的长教程不会伪装成已本地化页面，而是回退到英文内容并使用 noindex，降低重复内容和错误收录风险。

  • 修复

    修复多语言导航和内部链接回退

    侧边栏、顶部导航、搜索、翻页和底部链接会根据当前语言判断目标页面是否已翻译。未翻译的页面自动指向已有英文页面，避免生成大量不存在的本地化 404 链接。

  • 修复

    修复预渲染页面的 lang 与 RTL 属性

    静态预渲染阶段会根据 URL 前缀写入正确的 html lang 和 dir 属性，例如 pt-BR、id、ar、vi，保证静态页面被搜索引擎抓取时仍保留正确语言信号。

  • 优化

    多语言发布构建审计通过

    本次构建生成 409 个可收录 URL、675 条预渲染路由，并通过静态路由审计和 git diff --check，验证 sitemap、预渲染文件、canonical、语言入口和内部链接的一致性。
