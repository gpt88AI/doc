WorkRally Skills 总览

基于 Tencent/workrally 官方 Skill 与参考文档整理，完整说明 WorkRally CLI 的能力边界、安装方式、场景路由和各专项教程入口。

This page is accessible in English navigation, but the full body has not been translated yet. The original Chinese content is kept below for accuracy.
来源说明

本组页面基于 Tencent/workrally 的官方 Skill、README 与 references 文档整理而成。这里不是逐字搬运，而是面向 gpt88 文档站做了重组、补充和任务化改写。

这套 Skill 是什么

WorkRally CLI 是一套面向 AI Agent 的漫剧视频创作工具链。它不是单个“生图命令”，而是把项目、剧集、场次、素材上传、资产库、画布、AI 生图 和 AI 生视频 串成了完整工作流。

从使用者视角看，它解决的是“先有故事和角色，再批量做关键帧、动画、素材管理和画布排版”的问题， 比单纯的图片 API 或视频 API 更接近生产流。

核心能力

workrally-capabilitiestext
AI 生图
- Kontext 图片生成
- 多参考图输入
- 画布内自动占位节点

AI 生视频
- Text / FirstLastFrame / SubjectToVideo 三种驱动
- 动态模型与分辨率
- 任务轮询与批量生成

项目与场次
- project / series / shot 全流程 CRUD
- 角色识别、模型配置、结果查询

上传与素材
- upload → asset create → material add 三段式
- 媒资库与资产库分层管理

无限画布
- Yjs 协同
- 8 种节点类型
- build-draft 增量 / 删除 / 覆盖

安装与接入

quickstart.shbash
npm install -g workrally
workrally auth login
workrally auth status

# 先看自己要做哪类工作
# 1. 生图 / 生视频 → 先读 AI 生成指南
# 2. 上传素材 → 先读上传与素材管理
# 3. 做场次 / 漫剧 → 先读场次工作流
# 4. 操作画布 → 先读无限画布指南

官方 Skill 的入口是命令行工具 `workrally`。认证完成后，后续所有能力都通过该 CLI 访问，不需要你自己拼接大量后端接口。

先建立 4 个关键概念

概念作用常见误区
项目 project素材归属的上层容器。所有 asset 都必须属于项目，否则 Web 端不可见。把项目 ID 当成画布 ID 用。
画布 canvas无限画布空间，用于排版节点、放素材、放画板、文本和涂鸦。以为 canvas 和 project 是同一个东西。
媒资库 asset项目级文件池。上传完成后必须先入库，后续画布、资产库、生成能力都依赖它。上传后直接拿 CDN URL 当 asset_id 用。
资产库 material树形组织视图。把人物、道具、场景、网盘目录组织起来。以为上传文件后会自动出现在角色/道具/场景目录。

专项教程地图

  • AI 生成指南： 生图、生视频、画布自动占位、轮询、模型动态获取。
  • 上传与素材管理： `upload`、`asset create`、`material add` 的三段式流程。
  • 场次与剧集工作流： `series`、`shot`、角色识别、模型配置、批量生成和结果查询。
  • 无限画布指南： 8 种节点类型、画板规则、增量合并、覆盖模式。
  • 常见坑点与错误排查： 项目 / 画布混淆、URL 过期、模型硬编码、无效节点等高频问题。
