旧 gpt88.cc 账户迁移到 Agent

从旧 gpt88.cc 入口迁移到 Agent，完成登录、API Key 创建和第一次成功请求。

This page is accessible in English navigation, but the full body has not been translated yet. The original Chinese content is kept below for accuracy.
这篇教程只解决一个问题：从旧站正确迁移到新站

现在访问 gpt88.cc，页面会提示前往 agent.gpt88.cc 继续使用最新的账户、余额和 AI 服务。新手先完成迁移，再继续看 API Key、充值和工具接入教程。

先看结论

不要把“旧站还能打开”理解成“旧站仍然是所有操作的入口”。当前最稳妥的思路是：旧站负责告诉你迁移，新站负责账户和服务，文档站负责技术接入。

迁移完成的判断标准是：你能在新站注册或登录，能找到账户相关入口，并且知道下一步应该去哪个页面。 登录后的充值、创建 API Key、选择模型分组和 Codex 配置属于后续操作，按下面的顺序进行更容易排查问题。

5 步完成迁移

migration-flowtext
1. 打开旧站，确认是否看到“新站已上线”的迁移提示
2. 前往 https://agent.gpt88.cc
3. 新用户注册，老用户在新站登录
4. 登录后核对账户、余额和可用入口
5. 需要开发接入时，再进入 API Keys 和技术文档

迁移时不要同时排查模型、线路、客户端和 API Key。先确认站点入口正确，再做账户操作，最后才进入技术接入。

第一步：识别旧站迁移提示

打开 gpt88.cc 后， 如果看到顶部的“新站已上线”提示，就不要继续在旧站寻找新的注册或账户入口。 点击提示中的“前往新站”，或者直接打开 https://agent.gpt88.cc。

服务区域和条款以新站当前公告为准

新注册、充值和生成式 AI 服务可能受支持地区、服务条款或账户资格限制。遇到无法注册、无法充值或入口不可见时， 先查看新站当前公告和 支持地区 页面，不要反复刷新旧站。

第二步：进入 agent.gpt88.cc

新站首页会把大模型广场、价格对比、登录和图片工作台放在同一套入口里。你现在只需要记住两个按钮： 新用户点“免费注册使用”，已有账户点“登录”。

new-site-entriestext
新站首页       https://agent.gpt88.cc/home
注册入口       https://agent.gpt88.cc/register
登录入口       https://agent.gpt88.cc/login
API Keys       https://agent.gpt88.cc/keys      # 需要登录
模型广场       https://agent.gpt88.cc/model-square
价格对比       https://agent.gpt88.cc/pricing
技术文档       https://doc.gpt88.cc

第三步：新用户注册

打开 agent.gpt88.cc/register 注册页面，按页面要求填写邮箱和密码。当前注册页还可能提供可选优惠码。

  1. 输入你能正常收信的邮箱，后续账户验证和找回密码需要用到它。
  2. 设置至少 6 个字符的密码，不要复用邮箱密码或其他平台密码。
  3. 先打开并阅读最新服务条款、隐私政策和支持地区说明。
  4. 同意条款后，继续按钮和其他登录方式才会启用。
  5. 完成注册后，确认页面已进入新站账户或按提示完成邮箱验证。
为什么按钮一开始不能点？

当前页面在未同意最新条款前，会暂时禁用邮箱、密码和快捷登录控件。这是页面状态，不是浏览器或账号故障； 先阅读条款并按页面完成同意，再继续注册。

第四步：老用户登录

已有账户直接打开 agent.gpt88.cc/login 登录。不要把密码、验证码或 API Key 粘贴到公开文档、截图或聊天中。

  1. 确认地址栏是 agent.gpt88.cc，不是相似域名。
  2. 输入新站账户邮箱和密码。
  3. 如果登录页提示条款更新，先查看相关文档并按页面完成确认。
  4. 登录成功后，观察是否回到了你原本要访问的页面；例如访问 API Keys 时会回到 Key 管理页。

旧站和新站的账户状态不要凭感觉判断。登录成功后，以新站当前显示的账户、余额和可用入口为准； 如果无法登录，先用“忘记密码”流程或联系客服，不要连续创建多个重复账户。

登录后：按任务进入控制台

我已用登录后的控制台核对过以下入口。控制台页面会显示你的账户专属数据，因此教程只记录页面结构和操作顺序， 不展示任何用户名、余额、完整 API Key 或订单信息。

先充值，还是先创建 Key？

  1. 只想熟悉界面：先打开 仪表盘、模型广场和 价格对比，这些页面可以先用于认路和选型。
  2. 准备调用 API：先在 充值/订阅确认账户有可用额度， 再打开 API 密钥创建一个按项目命名的 Key。
  3. 准备配置 Codex：先完成 Key 创建和一次最短 API 请求，再按照 Codex CLI 接入配置工具。这样可以判断问题来自账户、Key、模型分组，还是本地配置。

API 密钥页面怎么看？

登录后打开 /keys，页面顶部有“创建密钥”、刷新、列设置、名称或 Key 搜索，以及分组和状态筛选。 页面同时给出两个常用端点：标准 API 使用 https://api.gpt88.cc，图片和视频等长任务使用 https://img.gpt88.cc。创建 Key 后，只在生成当下保存完整密钥；日后页面通常只显示脱敏值。

模型分组和用量怎么核对？

模型广场是公开目录， 可按平台、分组名、渠道名或模型名定位分组，并比较倍率、模型数量和模型范围。选定分组后，回到 API 密钥页面确认 Key 使用的分组， 再到 使用记录按时间、API Key、模型、分组、类型和计费类型筛选， 检查请求是否真正落到了预期线路。

登录后的页面不要直接截图公开

余额、用户名、订单和 API Key 都是账户数据。分享排障截图前，先遮住完整 Key、邮箱、用户标识、余额和支付信息； 复制端点时也要确认没有把认证请求头一起复制出来。

第五步：记住新站入口地图

入口负责什么新手怎么理解
gpt88.cc旧站入口 / 迁移公告看到迁移提示后，账户、余额和新服务以新站为准。

这里有一个容易混淆的点：agent.gpt88.cc 是账户和产品入口，api.gpt88.cc 是程序调用地址。不要把 API 地址当成浏览器登录页面，也不要把网页登录密码当成 API Key。

迁移后检查清单

完成注册或登录后，按下面顺序检查。每一步都应该得到一个明确结果：

  1. 账户：确认登录的是正确邮箱，个人菜单或账户区域可以正常打开。
  2. 余额：找到新站余额或计费入口；具体金额和可用套餐以当前控制台为准。
  3. 模型：打开模型广场，先按平台、分组名或模型名搜索，不要凭记忆手打模型 ID。
  4. 文档：需要 API 接入时打开 快速开始，不要从旧站公告反推参数。
  5. 工具：需要 Codex、Claude Code、Cursor 或 CC-Switch 时，进入对应的集成教程。

常见误区

  • 误区 1：旧站能打开，所以继续在旧站充值。先看是否有迁移提示；账户和余额操作以新站为准。
  • 误区 2：把 agent.gpt88.cc 当成 API Base URL。它是网站入口，标准 API 应使用 https://api.gpt88.cc。
  • 误区 3：把网页登录密码当成 API Key。API Key 要在登录后的 API Keys 页面单独创建和管理。
  • 误区 4：一上来就创建很多 Key。先完成迁移和账户确认，再按项目或工具用途创建独立 Key。
  • 误区 5：看到模型广场的公开分组就认为账号一定有权限。公开分组是选型参考，最终可用模型和权限以你的 Key/控制台为准。

下一步阅读顺序

迁移完成后，建议按下面顺序继续：

  1. 认证与计费：理解 API Key、余额、用量和分组倍率。
  2. 完整接入手册：从 Key、Base URL 到第一次请求。
  3. 配置文件导出：把 Key、模型和线路打包给不同工具。
  4. Codex CLI 接入：配置 API Key 模式并验证文件工具。

迁移完成的最小标准

你已经能在新站完成注册或登录，知道账户和 API 的入口分别在哪里，并能从文档站继续完成后续接入。 这时再做充值、创建 API Key 或配置 Codex，排障成本最低。

问题是什么

旧站入口、Agent 控制台和 API 端点不是同一个地址。迁移应先确认账户入口，再创建 API Key 并完成一次成功请求。

最短可用配置

.envbash
账户入口: https://agent.gpt88.cc
API 端点: https://api.gpt88.cc/v1
认证头: Authorization: Bearer <API_KEY>

API Key 只放在服务端环境变量或密钥管理器中，不要提交到 Git、截图、浏览器前端或公开 issue。

完整示例（curl / Python / Node.js）

request.shbash
curl https://api.gpt88.cc/v1/chat/completions \
  -H "Authorization: Bearer $OPENAI_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"model":"gpt-5.6-sol","messages":[{"role":"user","content":"返回 OK"}],"max_tokens":32}'
request.pypython
import os
from openai import OpenAI

client = OpenAI(
    api_key=os.environ["OPENAI_API_KEY"],
    base_url=os.getenv("OPENAI_BASE_URL", "https://api.gpt88.cc/v1"),
)
response = client.chat.completions.create(
    model=os.getenv("OPENAI_MODEL", "gpt-5.6-sol"),
    messages=[{"role": "user", "content": "返回 OK"}],
    max_tokens=32,
)
print(response.choices[0].message.content)
request.mjsjavascript
import OpenAI from "openai";

const client = new OpenAI({
  apiKey: process.env.OPENAI_API_KEY,
  baseURL: process.env.OPENAI_BASE_URL ?? "https://api.gpt88.cc/v1",
});
const response = await client.chat.completions.create({
  model: process.env.OPENAI_MODEL ?? "gpt-5.6-sol",
  messages: [{ role: "user", content: "返回 OK" }],
  max_tokens: 32,
});
console.log(response.choices[0].message.content);

常见错误

  • 登录成功但没有 Key：API Key 需要在 Agent API Keys 页面单独创建。
  • 401：不要把旧站密码当作 API Key。
  • 404：程序端使用 api.gpt88.cc/v1，不要请求 agent.gpt88.cc。
  • 余额或模型不一致：以新站当前账户和使用记录为准。

价格和计费说明

迁移不会自动承诺旧站套餐或余额继承。新站按人民币余额结算，实际扣费受模型、分组倍率和输入输出用量影响。
